# PFM and Feedback Concentration Model

This repository contains a MATLAB implementation of a phase-field model (PFM) and feedback concentration model of the paper: 

"Cell Migration Boundary Motion in Drosophila Egg Chambers: A Combined Phase Field and Chemoattractant Model"
Naghmeh Akhavan, et al., 2026


1. solve the baseline system without chemoattractant concentration to a steady state, and
2. from that steady state; evolve a dynamic chemoattractant concentration field and its coupled cluster response.

The repository also includes a visualization script for generating videos and figures from saved solver output.

## Overview

The model represents each cell as a smooth phase field on a two-dimensional Cartesian grid. The active cell ordering used throughout the code is (1) nurse cells (2) oocyte then (3) cluster.

The baseline solver evolves the cell phase fields under diffusion, adhesion, repulsion, and target-volume constraints. The concentration-coupled solver extends this system by:

- computing an effective diffusion coefficient from the current cell configuration,
- evolving a concentration field with secretion and decay,
- building receptor-driven tension and chemotactic forcing from concentration gradients, and
- applying those forces to the cluster phase field.

Details on the mathematical background can be found on the suggested citation.

## Repository Structure

- [config.m](/mnt/d/github/PFM_Dynamic_Concentration/config.m)
  Central configuration file. Defines geometry, model parameters, runtime flags, mesh metadata, and finite-difference operators. **Any user parameter can be controlled and changed via this script.**

- [go_solver.m](/mnt/d/github/PFM_Dynamic_Concentration/go_solver.m)
  Main execution script.

- [solve_pfm.m](/mnt/d/github/PFM_Dynamic_Concentration/solve_pfm.m)
  Baseline solver without concentration coupling.

- [solve_pfm_concen.m](/mnt/d/github/PFM_Dynamic_Concentration/solve_pfm_concen.m)
  Concentration-coupled system solver.

- [go_show_results.m](/mnt/d/github/PFM_Dynamic_Concentration/go_show_results.m)
  Visualization script for generating plots and videos from saved solver output.

- [psuedoboundaryconditiondiffusion.m](/mnt/d/github/PFM_Dynamic_Concentration/psuedoboundaryconditiondiffusion.m)
  Example 1-D diffusion simulation with moving pseudo-boundary.

- [initialize_u.m](/mnt/d/github/PFM_Dynamic_Concentration/initialize_u.m)
  Helper used to build the initial phase-field profile for each cell.

- [results/](/mnt/d/github/PFM_Dynamic_Concentration/results)
  Default output directory for solver-generated `.mat` files.

- [visuals/](/mnt/d/github/PFM_Dynamic_Concentration/visuals)
  Default output directory for figures and videos generated by `go_show_results.m`..

## Main Workflow

The normal workflow is:

1. configure the model in `config.m`
2. run `go_solver.m`
3. inspect or visualize the saved output with `go_show_results.m`

### Step 1: Configure The Model

Edit [config.m](/mnt/d/github/PFM_Dynamic_Concentration/config.m) to set:

- geometry and target-volume data for cells
- phase-field interaction parameters
- spatial and temporal discretization
- concentration-coupled parameters
- runtime flags controlling which solver stages are executed

The most important runtime flags are:
- `params.run_steadyQ`
  Set to `1` to run the steady-state phase-field solver.

- `params.run_concenQ`
  Set to `1` to run the concentration-coupled restart solve.


### Step 2: Run The Solvers

Run [go_solver.m](/mnt/d/github/PFM_Dynamic_Concentration/go_solver.m).

The script performs the following steps:

1. loads `config.m`
2. optionally runs `solve_pfm.m`
3. saves the steady-state results to `results/results.mat`
4. reloads the steady-state results
5. optionally runs `solve_pfm_concen.m`
6. saves the concentration-coupled results to `results/concentrationsolsaved.mat`

### Step 3: Visualize The Results

Edit the user options at the top of [go_show_results.m](/mnt/d/github/PFM_Dynamic_Concentration/go_show_results.m), then run the script.

The visualization script:

- loads either `results/results.mat` or `results/concentrationsolsaved.mat`
- validates that the expected variables are present
- writes figures or videos into `visuals/`

## Solver Outputs

### Steady-State Output

The steady-state solver writes `results/results.mat`, which contains at least:

- `fd` — finite-difference structure
- `params` — user defined model parameters
- `dof` — degree-of-freedom
- `sol_cells_saved` — saved cell phase-field states
- `residuals` — solver residual history

### Concentration-Coupled Output

The concentration-coupled solver writes `results/concentrationsolsaved.mat`, which contains at least:

- `fd` — finite-difference structure
- `params` — user defined model parameters
- `dof` — degree-of-freedom
- `sol_cells_saved_c` — saved cell phase-field states
- `residuals_c` — solver residual history
- `concen` — computed chemoattractant concentration field over time.
- `diff_coef` — effective diffusion coefficient field over time.
- `receptor` — receptor field over time.
- `chemo` — chemotactic forcing field over time.
- `tension_x` — x-component of receptor-driven tension force vector.
- `tension_y` — y-component of receptor-driven tension force vector.

## Recommended Usage

To run initial cells to Steady state then include concentration dynamics by setting following in `config.m`:

```matlab
params.run_steadyQ = 1;
params.run_concenQ = 1;
```

Then run:

```matlab
go_solver
```

Visualize desided plots. For example the cluster movement under the concentration dynamics can be shown by setting following in `go_show_results.m`:

```matlab
use_concentration = 1;
type_visual = 2;
type_f = 1;
```

Then run:

```matlab
go_show_results
```


## Troubleshooting (Common Errors)

### `results.mat` not found

Run the steady-state solver first:

```matlab
params.run_steadyQ = 1;
```

Then execute `go_solver.m`.

### `concentrationsolsaved.mat` not found

Enable and run the concentration-coupled solve:

```matlab
params.run_concenQ = 1;
```

### A visualization mode errors when `use_concentration = 0`

Some visualization modes require concentration output. Use:

- `type_visual = 1, 2, 3` with `type_f = 1` for steady-state cell visualization, or
- run the concentration-coupled solver first and set `use_concentration = 1`

### The time discretization assertions fail in `config.m`

Check that:

- `x_length / h_space + 1` is an integer
- `y_length / h_space + 1` is an integer
- `t_final / h_time` is an integer
- `t_final_c / h_time_c` is an integer
- `dof.n_saved` divides the steady-state step count
- `dof.n_saved_c` divides the concentration step count
